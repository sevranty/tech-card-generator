# tech-card-generator

`tech-card-generator` — публичный репозиторий личного Agent Skill **`machinery-card-generator`**. Навык анализирует ровно одну главную машину, при необходимости находит один реальный аналог, показывает полный русскоязычный анализ и создаёт проверенную образовательную карточку 2×1 для ребёнка до четырёх лет.

## Что делает навык

Навык принимает фотографию, изображение, визуальный референс, игрушку, рисунок, набросок, схему, стилизацию или текстовое описание техники и последовательно:

1. выбирает одну главную машину;
2. отделяет реальные признаки от игрушечных или стилизованных искажений;
3. при необходимости определяет один наиболее достоверный реальный класс техники;
4. готовит полный предметный анализ и четыре точных русских текста карточки;
5. показывает анализ пользователю до генерации;
6. фиксирует все данные без права самовольного изменения;
7. создаёт чёрно-белую образовательную карточку 2×1;
8. проверяет механику, композицию, действие, тексты и пользовательскую выдачу;
9. перегенерирует карточку при любой критической ошибке;
10. прикрепляет только последнюю версию, прошедшую проверку.

## Для кого

- родители и педагоги, создающие материалы для детей до четырёх лет;
- авторы детских энциклопедий и обучающих карточек;
- дизайнеры серийных иллюстраций о технике;
- пользователи Agent Skills, которым нужен воспроизводимый анализ и генерация изображения по строгому процессу.

## Поддерживаемые входы

- реальная фотография техники;
- иллюстрация или визуальный референс;
- игрушечная или упрощённая модель;
- детский рисунок или набросок;
- схема или стилизация;
- частично закрытый или плохо различимый объект;
- текстовое описание машины.

Навык всегда выбирает **ровно одну** главную машину. Остальная техника в кадре игнорируется и не переносится в итоговую карточку.

## Результат

### Этап А — видимый анализ до генерации

Пользователь получает:

- тип исходного материала;
- четыре варианта названия машины;
- один подтверждённый реальный аналог, если он необходим;
- видимые признаки, ограничения и уровень уверенности;
- функцию и точную рабочую часть;
- три реальные задачи;
- рабочее окружение;
- положение рабочей части, контакт с материалом и видимый результат действия;
- разные названия машины для левого и правого блоков;
- четыре точных русских текста карточки;
- предгенерационную проверку и подтверждение, что генерация ещё не запускалась.

### Этап Б — финальная карточка

После генерации и полной проверки пользователь видит только последнюю успешную карточку:

- горизонтальный формат 2×1;
- два равных квадратных блока;
- одна механически достоверная машина;
- наглядное рабочее действие;
- передний, средний и дальний планы;
- чёрно-белая техническая иллюстрация;
- четыре зафиксированных русских текста;
- отсутствие другой техники, людей, животных, брендов, моделей и псевдотекста.

## Визуальный принцип

Главная машина занимает примерно 50–60% безопасной области левого квадрата. Для окружения остаётся не менее 30% доступной площади. Фон на 20–35% светлее и проще машины, но не исчезает. Локальная зона чистой белой бумаги отделяет силуэт от фона без серой тени, свечения или эффекта наклейки.

Рабочая задача должна читаться без подписи: рабочая часть соприкасается с материалом, положение механически правдоподобно, а результат действия виден на изображении.

## Требования

Для полного выполнения навыка необходимы:

- клиент с поддержкой Agent Skills;
- визуальный анализ изображений;
- генератор изображений;
- возможность открыть и проверить созданный файл;
- возможность прикрепить итоговое изображение пользователю;
- веб-поиск по надёжным источникам, когда визуальных знаний недостаточно для выбора реального аналога.

Локальная проверка репозитория требует Python 3 и `PyYAML`:

```bash
python3 -m pip install pyyaml
```

## Установка

### Codex: навык для текущего репозитория

Скопируйте или создайте символьную ссылку на папку навыка в `.agents/skills`:

```bash
mkdir -p .agents/skills
cp -R machinery-card-generator .agents/skills/machinery-card-generator
```

### Codex: личный навык

```bash
mkdir -p "$HOME/.agents/skills"
cp -R machinery-card-generator "$HOME/.agents/skills/machinery-card-generator"
```

После добавления навыка перезапустите клиент, если он не появился автоматически.

Формат репозитория следует открытому стандарту Agent Skills: обязательный `SKILL.md`, дополнительные `references/`, `scripts/` и `agents/openai.yaml`.

## Вызов

### Явный вызов

```text
Используй machinery-card-generator полностью для приложенной фотографии техники.
```

В Codex CLI или IDE навык можно выбрать через `/skills` или упомянуть как `$machinery-card-generator`.

### Пример с референсом

```text
Используй machinery-card-generator. Выбери одну главную машину, покажи полный анализ и точные тексты до генерации, затем создай и проверь карточку 2×1.

```

## Обязательный workflow

Навык выполняет 21 этап без перестановки:

1. определяет тип входа;
2. выбирает одну машину;
3. определяет характер изображения;
4. при необходимости выбирает один реальный аналог;
5. проводит полный анализ;
6. готовит четыре названия;
7. выбирает разные названия для двух блоков;
8. готовит четыре точных текста;
9. показывает анализ пользователю;
10. выполняет предгенерационную проверку;
11. фиксирует данные;
12. формирует промпт только из зафиксированных данных;
13. создаёт изображение;
14. определяет актуальный путь файла;
15. проверяет существование и доступность;
16. открывает изображение и проводит визуальную проверку;
17. исправляет только найденную ошибку;
18. повторяет генерацию и проверку до устранения критических ошибок;
19. прикрепляет последнюю успешную версию;
20. проверяет отображение изображения в чате;
21. только после этого завершает задачу.

## Ключевые ограничения

Навык не должен:

- анализировать несколько машин одновременно;
- добавлять другую технику в фон;
- переносить игрушечные или фантастические искажения в реальную конструкцию;
- придумывать марку, модель, год, мощность, массу или другие характеристики;
- менять зафиксированные названия, тексты, действие, материал или окружение;
- использовать одинаковое название машины слева и справа;
- показывать статичную машину без понятного взаимодействия с материалом;
- создавать пустой студийный фон;
- принимать изображение с обрезанной машиной или текстом;
- завершать задачу пустым ответом или служебным результатом без видимого изображения.

Если нельзя надёжно определить даже общий класс техники, навык останавливается до генерации и запрашивает дополнительный референс.

## Проверенный пример: гусеничный бульдозер

**Вход:** реальная фотография одного гусеничного бульдозера с широким передним рабочим щитом.

**Выбор:** одна машина — гусеничный бульдозер.

**Рабочая задача:** передний отвал опущен к земле, входит нижней кромкой в рыхлый грунт и толкает его вперёд. Перед щитом видна сдвинутая земля, позади — более ровная полоса.

**Точные тексты карточки:**

- слева: «Гусеничный бульдозер разравнивает землю»;
- справа: «Бульдозер»;
- детский текст: «Бульдозер толкает землю большим щитом. Земля становится ровной.»;
- подпись детали: «Широкий щит».

**Проверки:** машина занимает 50–60% безопасной области, окружение занимает не менее 30%, присутствуют три плана, отвал соприкасается с грунтом, результат разравнивания виден, другой техники нет.

Полная фикстура: [`machinery-card-generator/examples/tracked-bulldozer.md`](machinery-card-generator/examples/tracked-bulldozer.md).

Полный dry run: [`machinery-card-generator/tests/dry-run-real-bulldozer.md`](machinery-card-generator/tests/dry-run-real-bulldozer.md).

## Структура репозитория

```text
tech-card-generator/
|-- README.md
|-- LICENSE
`-- machinery-card-generator/
    |-- SKILL.md
    |-- agents/
    |   `-- openai.yaml
    |-- references/
    |   |-- analysis-rules.md
    |   |-- card-layout.md
    |   |-- output-template.md
    |   |-- quality-check.md
    |   `-- style-lock.md
    |-- examples/
    |-- scripts/
    `-- tests/
```

`SKILL.md` содержит обязательный порядок выполнения. Подробные правила распределены по справочным файлам: анализ, стиль, компоновка, пользовательская выдача и контроль качества.

## Валидация

Из корня репозитория:

```bash
python3 machinery-card-generator/scripts/validate-agent-skills-spec.py machinery-card-generator
python3 machinery-card-generator/scripts/validate-content.py
```

Ожидаемый результат:

```text
Agent Skills specification validation passed
Content validation passed
```

Валидаторы проверяют frontmatter, имя папки, обязательные файлы, `agents/openai.yaml`, 21 этап, обязательные поля, ограничения масштаба, разные названия двух блоков, тестовые сценарии и правила видимого прикрепления финального изображения.
