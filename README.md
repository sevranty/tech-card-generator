# tech-card-generator

`tech-card-generator` — публичный репозиторий личного Agent Skill **`machinery-card-generator`**. Навык анализирует ровно одну главную машину, при необходимости выбирает один реальный аналог, показывает полный русскоязычный анализ и создаёт проверенную образовательную карточку 2×1 для ребёнка до четырёх лет.

## Что делает навык

Навык принимает фотографию, изображение, референс, игрушку, рисунок, набросок, схему, стилизацию или текстовое описание техники и последовательно:

1. выбирает одну главную машину;
2. отделяет реальные признаки от игрушечных или стилизованных искажений;
3. при необходимости определяет один наиболее достоверный реальный класс техники;
4. готовит полный анализ и четыре точных русских текста;
5. показывает анализ до генерации и фиксирует данные;
6. создаёт чёрно-белую карточку 2×1;
7. проверяет механику, композицию, действие, тексты и пользовательскую выдачу;
8. перегенерирует карточку при критической ошибке;
9. прикрепляет только последнюю версию, прошедшую проверку.

## Для кого

- родители и педагоги, создающие материалы для детей до четырёх лет;
- авторы детских энциклопедий и обучающих карточек;
- дизайнеры серийных иллюстраций о технике;
- пользователи Agent Skills, которым нужен строгий воспроизводимый workflow.

## Поддерживаемые входы

- реальная фотография техники;
- иллюстрация или визуальный референс;
- игрушечная или упрощённая модель;
- детский рисунок или набросок;
- схема или стилизация;
- частично закрытый или плохо различимый объект;
- текстовое описание машины.

Навык всегда выбирает **ровно одну** главную машину. Остальная техника игнорируется и не переносится в итоговую карточку.

## Результат

### Этап А — видимый анализ до генерации

Пользователь получает тип исходника, четыре названия, реальный аналог при необходимости, основания и ограничения, функцию и рабочую часть, три задачи, рабочее окружение, положение рабочей часта, контакт с материалом, видимый результат, разные названия для двух блоков, четыре точных текста и предгенерационную проверку.

### Этап Б — финальная карточка

После генерации пользователь видит только последнюю успешную карточку:

- горизонтальный формат 2×1 из двух равных квадратов;
- одна механически достоверная машина;
- наглядное рабочее действие;
- передний, средний и дальний планы;
- чёрно-белая техническая иллюстрация;
- четыре зафиксированных русских текста;
- отсутствие другой техники, людей, животных, брендов, моделей и псевдотекста.

## Визуальный принцип

Главная машина занимает примерно 50–60% безопасной области левого квадрата. Для окружения остаётся не менее 30% доступной площади. Фон на 20–35% светлее и проще машины, но не исчезает. Локальная зона чистой белой бумаги отделяет силуэт от фона без серой тени, свечения или эффекта наклейки.

Рабочая задача должна читаться без подписи: рабочая часть соприкасается с материалом, положение механически правдоподобно, результат действия виден.

## Требования

Для полного выполнения нужны клиент с поддержкой Agent Skills, визуальный анализ, генератор изображений, возможность открыть конечный файл и прикрепить изображение пользователю. Веб-поиск используется, когда визуальных знаний недостаточно для выбора реального аналога.

Локальная проверка требует Python 3, `PyYAML` и `Pillow`:

```bash
python3 -m pip install -r requirements-validation.txt
```

## Установка

### Навык текущего репозитория

```bash
mkdir -p .agents/skills
cp -R machinery-card-generator .agents/skills/machinery-card-generator
```

### Личный навык

```bash
mkdir -p "$HOME/.agents/skills"
cp -R machinery-card-generator "$HOME/.agents/skills/machinery-card-generator"
```

После добавления перезапустите клиент, если навык не появился автоматически.

## Вызов

```text
Используй machinery-card-generator полностью для приложенной фотографии техники.
```

В Codex CLI или IDE навык можно выбрать через `/skills` или упомянуть как `$machinery-card-generator`.

## Обязательный workflow

Навык выполняет 21 этап без перестановки:

1. определяет тип входа;
2. выбирает одну машину;
3. определяет характер изображения;
4. при необходимости выбирает один реальный аналог;
5. проводит полный анализ;
6. готовит четыре названия;
7. выбирает разные названия для двух блоков;
8. готовит четыре точных текста;
9. показывает анализ пользователю;
10. выполняет предгенерационную проверку;
11. фиксирует данные;
12. формирует промпт только из зафиксированных данных;
13. создаёт изображение;
14. определяет актуальный путь файла;
15. проверяет существование и доступность;
16. открывает изображение и проводит визуальную проверку;
17. исправляет только найденную ошибку;
18. повторяет генерацию и проверку;
19. прикрепляет последнюю успешную версию;
20. проверяет отображение изображения в чате;
21. только после этого завершает задачу.

Полный порядок находится в [`machinery-card-generator/SKILL.md`](machinery-card-generator/SKILL.md).

## Ключевые ограничения

Навык не должен анализировать несколько машин, добавлять другую технику, переносить игрушечные искажения, придумывать характеристики, менять зафиксированные данные, использовать одинаковые названия слева и справа, показывать статичную сцену, создавать пустой студийный фон, принимать обрезанное изображение или завершаться без видимой карточки.

Если нельзя определить даже общий класс техники, навык останавливается до генерации и запрашивает дополнительный референс.

## Проверенный пример

Для гусеничного бульдозера зафиксированы:

- слева: «Гусеничный бульдозер разравнивает землю»;

## Валидация

Полный локальный gate из корня репозитория:

```bash
python3 -m pip install -r requirements-validation.txt
python3 machinery-card-generator/scripts/validate-repository.py
```

Gate последовательно проверяет спецификацию Agent Skills, содержательные инварианты, обязательные файлы, относительные Markdown-ссылки, runtime manifest и воспроизводимый runtime harness. Успешный результат заканчивается строкой:

```text
Repository validation gate passed
```

Отдельные проверки:

```bash
python3 machinery-card-generator/scripts/validate-agent-skills-spec.py machinery-card-generator
python3 machinery-card-generator/scripts/validate-content.py
python3 machinery-card-generator/tests/runtime/run-runtime-validation.py
```

GitHub Actions запускает тот же gate для pull request и push в `main`.
