# TCG#10 — end-to-end runtime validation

## Итог

**PASS в границах зафиксированного runtime harness.** Проверены четыре сценария, порядок «видимый анализ → фиксация → генерация/остановка → открытие файла → визуальная проверка → контракт прикрепления» и неизменность четырёх текстов.

## Честная граница проверки

Среда редактирования не предоставляет API, которое запускает установленный личный skill как отдельный объект. Поэтому тест не называется нативным запуском ChatGPT Skill. Использован воспроизводимый deterministic contract harness: он берёт зафиксированные входы, формирует полный видимый анализ, блокирует неопределимый сценарий, создаёт проверяемые карточки и валидирует конечные файлы.

Не проверена стохастическая устойчивость конкретной модели генерации изображений. Она остаётся runtime-зависимостью, которую сам skill обязан обрабатывать циклом визуальной проверки и перегенерации.

## Сценарии

| Сценарий | Решение | Реальный класс | Результат |
|---|---|---|---|
| real-bulldozer | генерация | гусеничный бульдозер | PASS |
| toy-loader | один реальный аналог | компактный колёсный погрузчик | PASS |
| child-road-roller | один реальный аналог | дорожный каток | PASS |
| unclear-object | остановка до генерации | не определён | PASS |

## Проверено

- полный анализ расположен до раздела генерации;
- четыре названия и четыре точных текста зафиксированы;
- названия машины слева и справа различаются;
- для игрушки и рисунка выбран ровно один реальный аналог;
- неясный объект не запускает генерацию;
- три финальных файла имеют размер 1600×800;
- карточки состоят из двух равных квадратных блоков;
- одна полная машина находится только в левом блоке;
- справа показана только отдельная рабочая часть;
- присутствуют передний, средний и дальний планы;
- действие и видимый результат зафиксированы;
- нет людей, животных, другой техники, логотипов, моделей и псевдотекста;
- пользовательская выдача указывает только на последнюю проверенную версию.

## Команда воспроизведения

```bash
python3 -m pip install pillow
python3 machinery-card-generator/tests/runtime/run-runtime-validation.py
```

## Артефакты

- `runtime-manifest.json` — машинно-проверяемый результат;
- `run-runtime-validation.py` — воспроизводимый генератор и валидатор;
- `runtime-contact-sheet.svg` — сводная визуальная проверка трёх карточек и stop-case.
